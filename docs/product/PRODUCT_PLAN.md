# TokToTable --- Product Plan

## Purpose

TokToTable is a local-first weekly shopping tool that turns short-form
food content into structured recipes and a calm, reliable shopping list.

The product optimizes for **repetition, clarity, and user control**, not
automation or social engagement.

------------------------------------------------------------------------

## Core Principles

-   Local-first by default
-   No silent automation
-   User stays in control
-   Shopping List is the core product
-   AI parses, user decides
-   Stability before scale
-   Utility over engagement

------------------------------------------------------------------------

## Product Positioning

TokToTable lives **after TikTok**, not inside it.

It is not: - a social network - a creator platform - an AI demo - a
content feed

It is: - a personal food admin tool - a weekly ritual - a calm interface
for decisions

------------------------------------------------------------------------

## Product Phases

### V1 --- Local‑First Weekly Tool (Current)

**Status:** Complete

**Key Characteristics** - No login required - Fully usable without
account - Shopping List v2 is the home - Inline prices, pantry, aisles -
Keyboard-first flow - Explicit actions only

**Data (local only)** - Prices - Pantry - UI preferences - Shopping
lists - Recipes

------------------------------------------------------------------------

### V2 --- Account & Sync (Next)

**Goal:** Trust and continuity\
**Monetization:** None

**What login adds** - Sync across devices - Backup - Confidence

**What is synced** - Prices - Pantry - UI preferences

**What stays local** - Shopping lists - Recipes - History

**UX Rules** - Login is optional - No blocking flows - No forced
signup - Clear "local vs synced" state

------------------------------------------------------------------------

### V3 --- Pro / Plus

**Goal:** Time savings over time\
**Pricing:** €3--5 / month

**Unlocked Features** - Shopping history - Cost trends - Weekly/monthly
insights - Export (CSV / PDF) - Household profiles (later)

**Paywall Timing** - Only after multiple weeks of usage - Triggered by
desire for historical insight - Never blocks core functionality

------------------------------------------------------------------------

## Landing Page Strategy

### Purpose

-   Explain value clearly
-   Let users try without friction
-   Build trust

### Structure

-   Clear hero statement
-   Problem recognition
-   Simple "how it works"
-   Calm CTA: Try for free
-   Honest explanation of accounts

### No

-   Feature lists
-   AI hype
-   Community promises

------------------------------------------------------------------------

## App Shell

### Logged Out

-   Landing
-   How it works
-   Login
-   Try free

### Logged In

-   Shopping
-   Recipes
-   Settings
-   Account

Two worlds, one product.

------------------------------------------------------------------------

## Auth & Sync UX Contract

-   Account is a service, not a gate
-   Data remains accessible on logout
-   Sync is transparent and limited
-   Conflicts resolved via last-write-wins
-   No hidden background behavior

------------------------------------------------------------------------

## Explicit Non‑Goals

TokToTable will not become: - a recipe social feed - a creator
marketplace - an AI recommendation engine - a grocery marketplace

------------------------------------------------------------------------

## Long‑Term Vision

TokToTable should feel: - boring in the best way - predictable - calm -
trustworthy

The success metric is simple: \> "I use this every week without
thinking."

------------------------------------------------------------------------

## Guiding Sentence

**Stability before speed.\
Ritual before optimization.\
Utility before growth.**
